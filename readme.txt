Телеграм-бот для отслеживания билетов на поезда по Беларуси (pass.rw.by)
Пользователь может:

    Ввести город отправления, прибытия и дату поездки

    Получить список доступных поездов

    Выбрать поезд и просмотреть количество свободных мест по классам

    Начать отслеживание поезда — бот проверяет наличие билетов каждые 10 секунд

    Получать обновления при изменении количества мест

    Остановить отслеживание конкретного поезда или всех поездов

    Посмотреть список отслеживаемых поездов

Используемые технологии:

    python-telegram-bot через pyTelegramBotAPI

    requests, BeautifulSoup, threading

Ключевые особенности:
Асинхронность с threading: каждый отслеживаемый поезд запускается в отдельном потоке.

Хранение данных по chat_id: вся сессия пользователя сохраняется в user_data.

Работа с BeautifulSoup и SoupStrainer — эффективно парсит только нужные теги.

Гибкая обработка даты: поддержка разных форматов и валидация на "в прошлом" и "слишком далеко".

Вариант README - 2

1. Описание работы бота (для документации)
Telegram-бот для отслеживания железнодорожных билетов

Основной функционал:

Поиск доступных поездов между станциями

Отслеживание изменения количества билетов

Уведомления о появлении мест

Хранение истории запросов пользователей

Технические особенности:

Многопоточная архитектура (отслеживание поездов в фоне)

Работа с SQLite базой данных

Парсинг данных с сайта pass.rw.by

Логирование всех операций

Обработка ошибок и автоматические повторы

Команды бота:

/start - начать новый поиск

/stop - остановить бота и очистить данные

/show_track_list - показать отслеживаемые поезда

/stop_track_train - остановить отслеживание конкретного поезда

Рабочий процесс:

Пользователь вводит станции отправления и назначения

Указывает дату поездки

Получает список доступных поездов

Выбирает поезд для отслеживания

Бот проверяет наличие билетов каждые 5-10 минут

При изменениях отправляет уведомление
